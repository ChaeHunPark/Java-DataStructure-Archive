# 📅 Week 3: Graph (그래프) - 관계의 시각화와 탐색의 최적화

### 1. 핵심 개념 리마인드
* **그래프(Graph)**: 정점(Vertex)과 간선(Edge)의 집합. 트리와 달리 계층이 없고 **사이클(Cycle)**이 존재할 수 있는 가장 자유로운 비선형 자료구조.
* **표현의 두 얼굴**:
    - **인접 행렬(Matrix)**: $V \times V$ 배열. 연결 확인은 $O(1)$로 즉각적이나, 간선이 적을 때 공간 낭비($O(V^2)$)가 심한 '밀집(Dense)' 구조용.
    - **인접 리스트(List)**: 연결된 노드만 저장. 공간 효율이 극대화되어 대부분의 탐색 알고리즘에서 표준으로 사용되는 '희소(Sparse)' 구조용.



### 2. 주요 로직 구현 포인트
* **탐색의 종결자 (Visited)**: 트리와 달리 왔던 길을 다시 갈 수 있는 그래프의 특성상, `boolean[] visited` 배열은 무한 루프를 막는 유일한 논리적 경계선임.
* **DFS (깊이 우선 탐색)**:
    - **전략**: 스택(Stack) 또는 재귀를 이용해 한 경로를 끝까지 파고든 후 복귀(Backtracking).
    - **핵심**: 미로 찾기처럼 경로의 존재 여부와 특징을 기록하는 데 유리.
* **BFS (너비 우선 탐색)**:
    - **전략**: 큐(Queue)를 이용해 시작점으로부터 거리가 같은 노드들을 계층적으로 탐색.
    - **핵심**: **최단 경로(Shortest Path)** 보장. 큐에서 데이터를 꺼낼 때 시프팅 비용이 없는 `LinkedList`를 선택하여 $O(1)$의 삭제 효율을 확보.



### 3. 성능 벤치마크 분석
*정점 5,000개, 간선 10,000개의 희소 그래프 환경 테스트 결과*

| 지표 | 인접 행렬 (Matrix) | 인접 리스트 (List) | 이유 |
| :--- | :--- | :--- | :--- |
| **탐색 시간** | 약 25.4 ms | **약 0.8 ms** | 행렬은 모든 인덱스($V^2$)를 전수 조사해야 함 |
| **공간 복잡도** | $O(V^2)$ (약 100MB) | **$O(V+E)$** (약 2MB) | 리스트는 실제 간선 데이터만 메모리에 할당 |
| **연산 효율** | 불필요한 0 확인 과다 | **실제 연결 노드 즉시 탐색** | 데이터 밀도에 따른 자원 활용 최적화 |



### 💡 Insight
"트리가 '부모-자식'이라는 명확한 질서 아래 단방향으로 흐르는 통제된 사회라면, 그래프는 어디로든 연결될 수 있는 자유로운 관계망이다. 이 자유로움 속에서 길을 잃지 않게 해주는 **`visited` 배열**은 탐색의 안정성을 지탱하는 생명줄과 같다.

성능 테스트를 통해 확인한 **인접 리스트의 압도적 우위**는, '모든 가능성(Matrix)'을 열어두는 것보다 '실제 연결된 사실(List)'에 집중하는 것이 얼마나 강력한 효율을 내는지 증명한다. 특히 BFS 구현 시 `ArrayList` 대신 `LinkedList`를 큐로 선택하여 $O(1)$의 삭제 성능을 확보한 것은, 알고리즘의 논리만큼이나 **기저 자료구조의 물리적 특성**을 이해하는 것이 최적화의 핵심임을 시사한다."

## 🔥 핵심 인사이트
> "트리가 '질서'라면 그래프는 '관계'다. 그래프 탐색에서 **`visited` 배열**은 무한 루프를 막는 논리적 경계선이자, '가본 길'과 '가야 할 길'을 구분하는 유일한 기록 장치가 된다. 성능 테스트를 통해 확인한 인접 리스트의 우위는 모든 가능성(Matrix)을 열어두는 것보다 실제 연결된 사실(List)에 집중하는 것이 실전에서 얼마나 강력한지 증명해주었다. 결국 알고리즘의 우아함만큼이나 중요한 것은 데이터를 담는 그릇의 물리적 성질임을 체감한 주차였다."
