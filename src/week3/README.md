# 🏁 WEEK3: 비선형 자료구조 (Non-Linear Structures - Tree & Graph)

3주차에는 데이터 간의 계층 구조를 나타내는 **트리(Tree)**와 복잡한 네트워크 관계를 표현하는 **그래프(Graph)**를 학습했습니다. 재귀적 사고를 통해 탐색 효율을 극대화하고, 다양한 구현 방식(Matrix vs List)에 따른 성능 임계점을 분석했습니다.

## 📌 학습 로드맵
1. **Binary Search Tree**: 엄격한 좌우 정렬 규칙을 통한 $O(\log n)$ 탐색 및 재귀적 구조 설계
2. **Successor Logic**: 삭제 연산 시 트리의 정렬 상태를 유지하기 위한 후계자(Successor) 탐색 알고리즘
3. **Graph Representations**: 인접 행렬(Matrix)과 인접 리스트(List)의 구현 및 공간/시간 복잡도 비교
4. **Traversal Algorithms**: DFS(깊이 우선)와 BFS(너비 우선)를 활용한 비선형 데이터 완전 탐색

---

## 📊 자료구조별 특징 및 시간 복잡도 ($O$)

| 자료구조 | 주요 연산 | 시간 복잡도 | 특징 |
| :--- | :--- | :---: | :--- |
| **BST (Tree)** | `add()`, `remove()` | $O(\log n)$ | 데이터의 크기에 따른 계층적 정렬 및 효율적 동적 관리 |
| **Graph (List)** | `dfs()`, `bfs()` | $O(V + E)$ | 연결된 간선만 탐색하여 메모리 및 탐색 속도 최적화 |
| **Graph (Matrix)**| `dfs()`, `bfs()` | $O(V^2)$ | 간선 존재 여부를 $O(1)$에 확인 가능하나 자원 소모가 큼 |

> **주의**: BST는 트리가 한쪽으로 치우칠 경우 최악 $O(n)$까지 성능이 저하될 수 있음을 이해하고 균형 트리의 필요성을 인지함.

---

## 💡 주요 기술적 의사결정 (Key Decisions)

### 1. 트리 구조의 유지: Successor 전략 (tree/)
단순 노드 삭제보다 복잡한 '자식이 둘인 노드'의 삭제를 해결하기 위해 **오른쪽 서브트리의 최솟값**을 찾는 Successor 전략을 사용했습니다. 이를 통해 트리의 전체 구조를 파괴하지 않고 참조 연결만으로 안정적인 데이터 재배치를 구현했습니다.


### 2. 그래프 탐색의 종결성: visited 배열 (graph/)
그래프의 사이클(Cycle) 구조로 인한 무한 루프를 방지하기 위해 `boolean[] visited` 배열을 도입했습니다. 이는 DFS의 재귀 호출 깊이를 제어하고, BFS의 중복 큐 삽입을 막아 탐색의 안전성을 보장하는 핵심 장치가 되었습니다.

### 3. BFS 성능 최적화: 시프팅 비용 제거
너비 우선 탐색 구현 시 `ArrayList` 대신 `LinkedList`를 큐로 활용했습니다. 큐의 첫 번째 요소를 제거할 때 발생하는 시프팅 오버헤드를 제거하여 $O(1)$의 삭제 성능을 확보, 대규모 노드 탐색 시의 병목 현상을 해결했습니다.


### 4. 실증적 성능 분석: Matrix vs List
5,000개의 정점 환경에서 성능 테스트(`GraphPerformanceTest`)를 수행했습니다. 인접 리스트가 인접 행렬보다 약 **30배 이상** 빠른 결과를 데이터로 확인하며, 희소 그래프(Sparse Graph) 상황에서의 자료구조 선택 기준을 확립했습니다.
