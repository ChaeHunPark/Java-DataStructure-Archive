# ⏱️ 시간 복잡도(Time Complexity) 요약 가이드

이 문서는 본 저장소에서 구현된 주요 자료구조와 알고리즘의 성능 지표인 빅오(Big-O) 표기법을 정리한 가이드입니다.

---

## 📊 1. 자료구조별 시간 복잡도

| Data Structure | Access | Search | Insertion | Deletion | 특징 |
| :--- | :---: | :---: | :---: | :---: | :--- |
| **ArrayList** | $O(1)$ | $O(n)$ | $O(n)$ | $O(n)$ | 인덱스 접근은 빠르나, 삽입/삭제 시 시프팅 발생 |
| **LinkedList** | $O(n)$ | $O(n)$ | $O(1)$ | $O(1)$ | 삽입/삭제는 노드 연결만 바꾸면 되어 효율적 |
| **Stack (LIFO)** | - | - | $O(1)$ | $O(1)$ | 항상 상단(Top)에서만 데이터 처리가 일어남 |
| **Queue (FIFO)** | - | - | $O(1)$ | $O(1)$ | LinkedList 기반 구현 시 시프팅 비용 제거 |
| **HashMap** | - | $O(1)$ | $O(1)$ | $O(1)$ | 해시 충돌이 없을 시 상수 시간 내 처리 가능 |
| **BST (Tree)** | - | $O(\log n)$ | $O(\log n)$ | $O(\log n)$| 데이터 불균형 시 최악 $O(n)$ 성능 저하 주의 |
| **Trie** | - | $O(L)$ | $O(L)$ | $O(L)$ | 문자열 길이($L$)에만 의존하는 최적화 구조 |

## ⚡ 2. 알고리즘별 시간 복잡도

### 정렬 알고리즘 (Sorting)
| Algorithm | Best | Average | Worst | Space | 비고 |
| :--- | :---: | :---: | :---: | :---: | :--- |
| **Bubble Sort** | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 구현이 쉬우나 실무 사용 지양 |
| **Selection Sort** | $O(n^2)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 교환 횟수가 적음 |
| **Insertion Sort** | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | **거의 정렬된 상태에서 최강** |
| **Quick Sort** | $O(n \log n)$ | $O(n \log n)$ | $O(n^2)$ | $O(\log n)$ | 피벗 선택에 따라 성능 가변적 |
| **Merge Sort** | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ | **안정적이나 추가 메모리 필요** |

### 탐색 알고리즘 (Search)
| Algorithm | Time Complexity | 조건 / 특징 |
| :--- | :---: | :--- |
| **Linear Search** | $O(n)$ | 정렬되지 않은 데이터 전수 조사 |
| **Binary Search** | $O(\log n)$ | **반드시 정렬된 데이터**여야 함 |
| **DFS / BFS** | $O(V + E)$ | 그래프 모든 노드 방문 (인접 리스트 기준) |

## 📈 3. 성능 비교 그래프 (Concept)

데이터 크기($n$)가 커질수록 알고리즘 성능의 격차는 기하급수적으로 벌어집니다.

1. **$O(1)$**: 상수 시간 (최고의 효율)
2. **$O(\log n)$**: 로그 시간 (데이터가 커져도 성능 저하가 매우 적음)
3. **$O(n)$**: 선형 시간 (데이터 크기에 비례)
4. **$O(n \log n)$**: 선형 로그 시간 (효율적인 정렬 알고리즘의 표준)
5. **$O(n^2)$**: 이차 시간 (데이터가 많아지면 사용 불가 수준)

---

## 💡 Insight for Developers
- **$O(n^2)$ 정렬을 피하자**: 데이터가 10,000개만 넘어가도 퀵 정렬($O(n \log n)$)과의 성능 차이는 수십 배 이상 벌어집니다.
- **공간과 시간의 Trade-off**: 병합 정렬이나 트라이(Trie)처럼 메모리를 더 써서 시간을 단축하는 기법을 상황에 맞게 선택해야 합니다.
- **자료구조의 이해가 우선**: 알고리즘 성능은 결국 그 알고리즘이 사용하는 **자료구조의 물리적 특성**에 종속됩니다.
