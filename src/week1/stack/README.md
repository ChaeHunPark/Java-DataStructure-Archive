## 📅 Day 3: Stack (스택) - 후입선출(LIFO)의 설계

### 1. 핵심 개념 리마인드
* **LIFO (Last-In-First-Out)**: 가장 나중에 삽입된 데이터가 가장 먼저 추출되는 구조.
* **추상화와 위임(Delegation)**: 처음부터 새로 만드는 대신, 이미 검증된 `MyArrayList`를 내부 저장소로 사용하여 안정성을 높임.

### 2. 주요 로직 구현 포인트
* **저장소 선택 (ArrayList)**: 스택은 항상 한쪽 끝(Top)에서만 삽입과 삭제가 발생하므로, 인덱스 접근이 빠르고 맨 뒤 요소 처리가 $O(1)$인 배열 기반 리스트가 최적임.
* **주요 메서드**:
    - `push(E item)`: 리스트의 끝에 데이터를 추가 ($O(1)$).
    - `pop()`: 가장 마지막에 추가된 `size - 1` 인덱스의 요소를 제거하고 반환 ($O(1)$).
    - `peek()`: 요소를 제거하지 않고 가장 위의 데이터만 확인.
* **예외 처리**: 스택이 비어있을 때 `pop`이나 `peek` 시도 시 `RuntimeException`을 발생시켜 프로그램의 안정성 확보.

### 3. 시간 복잡도 분석 ($O$)
| 연산 | 복잡도 | 이유 |
| :--- | :--- | :--- |
| **push** | $O(1)$ | 배열 끝에 데이터 삽입 (Resizing 제외) |
| **pop** | $O(1)$ | 마지막 인덱스 데이터 삭제 및 시프팅 없음 |
| **peek** | $O(1)$ | 마지막 인덱스 즉시 조회 |

### 💡 Insight
"스택은 데이터의 '역순'을 다룰 때 강력하다는 것을 배웠다. 특히 `ArrayList`의 기능을 재활용(Composition)하면서, 사용자에게는 스택의 규칙(push/pop)만 노출시키는 인터페이스 설계의 중요성을 깨달았다."
